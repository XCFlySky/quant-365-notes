# ⑤ 避坑指南与实战规则

> 这些坑你不主动避开，迟早会主动找上你。每个坑都附了具体的避坑动作。

## 五个常见的坑

### ⚠️ 坑 1：不区分简单和对数，混着算夏普

**现象**：拿简单日收益的均值和标准差直接算夏普比率。
**为什么错**：简单收益不可加，数学上不成立。
**避坑动作**：**用对数收益算夏普，用简单收益做累计展示**——两个用途分清楚。

### ⚠️ 坑 2：短窗口硬年化

**现象**：用 1 个月数据 × 12 推年化，或 3 个月 × 4。
**为什么错**：短窗口里噪声远大于信号，年化出来的数字毫无意义。
**避坑动作**：**年化最小窗口是 1 年（252 个交易日）**。低于这个长度，只能老实讲「近期表现」，不要折算成年化。

### ⚠️ 坑 3：忘了分红

**现象**：用价格收益算夏普或长期回报。
**为什么错**：低估真实回报。高分红蓝筹股（银行、电力、REITs）的分红贡献能占 50% 以上。
**避坑动作**：**永远用复权价**。

### ⚠️ 坑 4：复权方式混用

**现象**：前复权数据和后复权数据混在同一套分析里。
**为什么错**：同一只股票不同时点的「同一价格」实际代表不同收益，数据自相矛盾。
**避坑动作**：**回测期内只用一种复权**（推荐后复权）；数据更新后整组重算。

### ⚠️ 坑 5：给 24 小时的市场用 252

**现象**：算比特币的年化波动率时用 √252。
**为什么错**：加密市场周末不关门，用 252 会低估真实波动。
**避坑动作**：**加密用 365，或按 8760 小时算**（BTC 波动率常按小时计）。这个错误在加密研报里非常常见。

## 收益率使用 SOP（打印贴在电脑边）

按这套规则走，你的执行力立刻拉到中等机构水平：

| # | 规则 |
| --- | --- |
| 01 | **面向人的报表展示**：用简单收益（直观）+ 累计净值曲线 |
| 02 | **统计推断 / 建模 / 夏普计算**：用对数收益 |
| 03 | **年化**：收益用 (1+R)^N − 1，波动率用 σ × √N，N 是交易日数不是 365 |
| 04 | **N 的取值**：美股 252、A 股 244、港股 247、加密 365 或更细 |
| 05 | **复权**：数据源固定后用同一种复权（推荐后复权）做整套研究 |
| 06 | **短窗口（< 60 日）绝对不年化**，只讲「近期表现」 |
| 07 | **别造轮子**：QuantStats / Pyfolio 是行业最佳实践，直接用比自己写靠谱 |
