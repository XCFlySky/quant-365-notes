# ② t 检验与多次检验陷阱

> t 检验就是把「信号 ÷ 噪声」算成一个数；而多次检验告诉你：测得多了，假阳性是数学必然。

## t 检验：三个输入，一个输出

t 检验用来回答「样本均值是否显著不同于零」。公式：

```text
t = (x̄ - μ₀) / (s / √n)      自由度 df = n - 1
```

三个输入：

| 输入 | 角色 | 直觉 |
| --- | --- | --- |
| x̄ − μ₀（样本均值 − 假设值） | 信号 | 收益偏离零有多远 |
| s（样本标准差） | 噪声 | 收益波动有多大 |
| n（样本量） | 放大镜 | 样本越多，噪声被压得越小 |

**t 统计量的本质 = 信号 / 噪声。** |t| 越大 → 信号越强或噪声越小 → 越显著。

查表对照（大样本近似）：

| t 值门槛 | 对应 p 值 | 意义 |
| --- | --- | --- |
| t > 1.96 | p < 0.05 | 学术「显著」底线 |
| t > 2.58 | p < 0.01 | 较显著 |
| t > 3.29 | p < 0.001 | 很显著 |
| t > 5.00 | p < 5×10⁻⁷ | Renaissance 标准 |

Python 里一行搞定：`scipy.stats.ttest_1samp(returns, popmean=0)` 直接返回 (t, p)。

## 算一个例子（课程样本）

策略 252 天日收益，样本均值 0.03%，样本标准差 1.0%，n=252：

```text
t = 0.0003 / (0.01 / √252) = 0.0003 / 0.00063 ≈ 0.48
```

t = 0.48 远小于 1.96 → p ≈ 0.32，不显著。想让 t 变大只有三条路：**提高均值（增 alpha）、降低波动（降 std）、拉长样本（更长回测）**。

## Type I / Type II 错误：两种「判错案」

| | H₀ 实际为真（策略没用） | H₁ 实际为真（策略有用） |
| --- | --- | --- |
| 你拒绝 H₀ | ❌ **Type I 错误（假阳性 α）**：以为有用，其实没用 | ✅ 正确发现 |
| 你保留 H₀ | ✅ 正确放过 | ❌ **Type II 错误（假阴性 β）**：错过一个真策略 |

p < 0.05 控制的是 α（假阳性率）。但很多人忘了 β 也重要：**检验功效（power = 1 − β）**是「策略真有用时，你能检测出来的概率」。样本太少时 power 可能只有 0.3——你不是没看到信号，是样本不够看不见。

## 多次检验：假阳性是数学必然

同时测 100 个策略，按 α=0.05，纯运气就会有约 **100 × 0.05 = 5 个假阳性**。这不是「可能」，是数学期望。

**p-hacking 的三种典型操作**：

1. 跑 1000 个不同参数的策略，只把 p 最小的拿出来报告；
2. 不停换样本期（2018–2020 不显著？换 2019–2021），直到出现 p < 0.05；
3. 加特征、删特征、调超参，直到结果好看。

> [!WARNING]
> Ioannidis 2005 年那篇著名论文标题就叫《Why Most Published Research Findings Are False》。你看到「统计显著」的策略或论文，很可能只是大量尝试中的**运气幸存者**。

## Bonferroni 修正：简单粗暴但有效

```text
Bonferroni 修正：α_corrected = α / m，m = 检验次数
```

举例（课程样本）：你跑 50 个因子做选股，每个都做 t 检验。按 α=0.05 会期望 2~3 个假阳性；Bonferroni 修正后阈值 = 0.05/50 = **0.001**，只有 t > 3.3 的因子才能进下一轮。

它很保守（可能误杀真信号），学术界也常用 BH-FDR（`statsmodels.stats.multitest.fdrcorrection`）折中。但作为散户第一道防线，Bonferroni 够用。

## 为什么学术界要降到 p < 0.005

2015 年心理学复现项目重做 100 篇顶刊论文，只有 36 篇能复现。0.05 太松，大量「显著结果」重现不了（replication crisis）。Benjamin 等人 2018 年在《Nature Human Behaviour》发文主张默认门槛降到 0.005。Renaissance 的 t > 5 比这还要严得多。

## 散户实战四条纪律（课程建议）

1. 在固定回测期（out-of-sample）只跑一次，不要换日期换参数；
2. 多策略比较时用 Bonferroni 或 FDR 修正；
3. 实盘前必须有 forward test（walk-forward）6 个月以上；
4. p 值不是唯一证据，还要看经济直觉。

> [!TIP]
> 一句话带走：**t 检验帮你算「信号/噪声」，Bonferroni 帮你管住「手贱多测」，两者都要成为肌肉记忆。**
