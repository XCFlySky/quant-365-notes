# Day 013 · 假设检验入门

> 你的策略回测年化 25%，是真本事还是运气？假设检验是量化里的「测谎仪」，它给你一个数字——p 值——来回答这个问题。

- 📂 阶段：P1 · 量化基础
- ⏱️ 时长：约 20 分钟
- 🎯 关键词：零假设、p 值、t 检验

## 这节课为什么重要

你回测一个策略，过去三年年化 25%，跑赢沪深 300 八个点——**这是真本事还是运气？** 没有假设检验，这个问题只能靠信仰回答。这节课讲清三件事：

1. **零假设 H₀ 和备择假设 H₁ 的对立结构**——先假设策略没用，看数据能不能推翻它；
2. **t 检验怎么算**——从样本均值、标准差、样本量算出 t 统计量，再查 p 值；
3. **p < 0.05 不是金科玉律**——样本量、多次检验、p-hacking 会让 0.05 失效，学术圈现在主张 p < 0.005。

学完你能用一个数字判断别人（或你自己）的策略是真本事还是赌运气——这是量化研究里「防止自欺欺人」的核心工具。

## 学习目标

- [x] 理解「零假设 H₀」和「备择假设 H₁」的对立结构，以及为什么先假设「没用」再去推翻
- [x] 掌握 t 检验的三个输入（样本均值 / 样本标准差 / 样本量）和输出（t 统计量 + p 值）
- [x] 会用 `scipy.stats.ttest_1samp` 一行做策略 t 检验，看 p 值判断显著性
- [x] 知道 p < 0.05 不是金科玉律——样本量大时几乎什么都显著，小时几乎什么都不显著
- [x] 理解多次检验陷阱（同时跑 100 个策略，5 个假阳性是必然的）以及 Bonferroni 修正
- [x] 认识 p-hacking 的真实危害，以及学术界 2017 年后主张降到 p < 0.005 的原因

## 一点历史八卦

- **1925 年**，英国统计学家 Ronald Fisher 在剑桥一个茶会上，听一位女士说她能凭口感判断奶茶是「先倒奶」还是「先倒茶」。Fisher 设计了实验：8 杯茶随机排序让她猜，全猜对的纯运气概率是 1/70 ≈ 1.4%——结果她真全对了。这就是假设检验的起源，也是 **p < 0.05 这个数字第一次被人为定下来**的地方；
- **1933 年**，Neyman 和 Pearson 提出 Type I/II 错误概念，完善了检验框架；
- **1988–2018 年**，Renaissance 的 Medallion 基金扣费前年化 66%，核心纪律之一：新策略上线前 t 统计量必须超过 5（p < 5×10⁻⁷），比学术标准严 100 万倍，30 年无年度亏损；
- **2005 年**，John Ioannidis 发表《Why Most Published Research Findings Are False》，揭开 p-hacking 的盖子；
- **2017 年后**，Andrew Gelman 等统计学家带头主张把显著性门槛降到 p < 0.005。

> [!TIP]
> 本节最重要的两句话：**先假设它没用，看数据能不能推翻；统计显著 ≠ 经济意义 ≠ 未来稳定。**
