# ① 零假设与 p 值

> 假设检验的套路很「绕」：不直接证明策略有用，而是先假设它没用，再看数据能不能把这个假设打脸。

## 为什么这么绕？黑天鹅的类比

想证明「天鹅都是白的」，你得看遍全世界所有天鹅——做不到。但只要看到**一只黑天鹅**，就能直接否定这句话。

假设检验玩的就是这个逻辑：**证明「X 为真」很难，找「X 为假」的反例容易**。所以我们不证明「策略有效」，而是假设「策略无效」，然后看数据有多支持这个反例。

## 两个假设的对立结构

| 假设 | 内容 | 白话 |
| --- | --- | --- |
| 零假设 H₀ | 策略平均收益 = 0 | 「这策略纯靠运气」 |
| 备择假设 H₁ | 策略平均收益 > 0（单尾）或 ≠ 0（双尾） | 「这策略真有 alpha」 |

我们的工作是看数据能否**拒绝 H₀**。

```text
H₀: μ = 0   vs   H₁: μ > 0 (单尾)  或  H₁: μ ≠ 0 (双尾)
```

> [!IMPORTANT]
> **拒绝不了 H₀ ≠ 策略没用。** 只能说「目前数据不足以证明它有用」——可能真没用，也可能有用但样本太小看不出来。这两者的区别，是新手最容易栽的地方。

## p 值：这节课最重要的一个数

**p 值的定义**：假设 H₀ 是真的（策略真的没用），那么**纯靠运气，至少出现这么好结果的概率**有多大。

举个例子（课程示例）：你的网格策略过去 252 天日均收益 +0.03%。

- H₀ 说：真实日均收益 = 0，这 +0.03% 是运气；
- 算出 p 值 = 0.32，意思是：就算策略真没用，纯运气也有 32% 的概率表现得比这还好；
- 32% 太高了，完全不算稀奇 → **无法拒绝 H₀**，你不敢上线。

如果 p 值 = 0.01：纯运气只有 1% 的概率撞出这成绩 → 要么是运气爆棚，要么 H₀ 是错的 → 拒绝 H₀，敢上线了。

## 经典误读对照表

| 误读 | 真相 |
| --- | --- |
| 「p 值 = H₀ 为真的概率」 | ❌ p 值是「H₀ 为真时，数据这么极端的概率」，条件搞反了 |
| 「p > 0.05 → 策略没用」 | ❌ 只是「数据不足」，可能是样本太小的锅 |
| 「p < 0.05 → 策略一定有用」 | ❌ 还有 5% 的假阳性率；多次检验时更糟（下一页讲） |
| 「p 越小，策略赚越多」 | ❌ p 只说「是不是运气」，不说「值不值得做」 |

> [!WARNING]
> p 值回答的是「**这是不是运气**」，不回答「**这值不值得做**」。一个年化 0.25% 的策略，只要样本够大也能 p < 0.05——但扣掉手续费你是亏的。统计显著和经济意义是两回事。

## 一张图记住流程

```text
提出 H₀（策略没用）
   ↓
收集数据（回测收益）
   ↓
算「H₀ 为真时，这么极端的概率」= p 值
   ↓
p < α（常用 0.05）？── 是 → 拒绝 H₀，策略「统计显著」
   └── 否 → 证据不足，继续攒数据或放弃
```

> [!TIP]
> 把 p 值想象成法庭判案：H₀ 是「被告无罪」，p 值是「如果被告真无辜，出现这些证据的概率」。概率太低，就合理怀疑被告不无辜——但永远不能说「百分百有罪」。
