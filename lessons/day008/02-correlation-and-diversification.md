# ② 皮尔逊 vs 斯皮尔曼与伪分散

> 相关系数有两种算法，一个怕极端值，一个不怕；而「分散」这件事，大多数人理解得都是错的。

## 皮尔逊相关：默认的那把尺子

**皮尔逊相关系数**（Pearson correlation）就是上一页那个标准化公式：

```text
ρ = Cov(X, Y) / (σ_X · σ_Y)
```

它度量的是**线性关系**：两只资产的收益点画在散点图上，越接近一条直线，|ρ| 越接近 1。它是所有软件的默认值（`df.corr()` 不加参数就是它），优点是快、直觉、可直接接进 wᵀΣw。

代价：**怕极端值**。一天 ±10% 的暴涨暴跌，平方项会主导整个协方差，把 ρ 拉高或压低。

## 斯皮尔曼秩相关：不看数值看排名

**斯皮尔曼相关系数**（Spearman correlation，也叫秩相关）的思路很妙：

1. 把两只资产的收益各自**换成排名**（收益第几高，而不是收益是多少）；
2. 再对排名算皮尔逊。

排名之后，「+10%」和「+0.5%」的差距被抹平——它们只差一个名次。所以斯皮尔曼**抗异常值**，而且能抓到「单调但不线性」的关系（比如一只资产涨得越多另一只也涨但越来越慢）。

课程实测（样本）：比亚迪 ↔ 宁德时代的皮尔逊是 **0.57**，斯皮尔曼是 **0.53**——两个值接近，说明这段数据里没有大量极端值在搅局。

> [!TIP]
> 课程给的判据：**当皮尔逊和斯皮尔曼的差距 > 0.15 时，警惕极端值在影响你的相关性估计**。这时候信斯皮尔曼更多一点。

## 伪分散：三个鸡蛋一个篮子

这节课的核心案例。两个对照组合（等权重，2 年日收益，课程样本）：

| | 组合 A：三只新能源车 | 组合 B：跨资产 |
| --- | --- | --- |
| 成分 | 比亚迪 + 宁德时代 + 特斯拉 | 比亚迪 + 中国石油 + 黄金 ETF |
| 内部相关性 | 高（比亚迪↔宁德 ρ=0.57） | 低（比亚迪↔黄金 ρ=0.03） |
| 组合年化波动 | 26.5% | 25.9% |

组合 A 看着有三只股票，其实是**同一只票的复制粘贴**：政策、产能、上游硅料这些驱动逻辑是同一套，一跌全跌。组合 B 的三只资产驱动逻辑完全不同（新能源车 / 石油 / 避险黄金），才是真正把鸡蛋放进了不同的篮子。

> [!IMPORTANT]
> 真分散不在于**数量**，在于**驱动逻辑的多样性**。检查你的组合：把所有持仓两两算相关系数，一堆 0.6+ 的话，你拥有的不是组合，是一只大号股票。

## 两个必须记住的警告

**警告一：线性相关 ≠ 因果。**

ρ=0.9 不代表 A 导致 B。可能两者都被第三个因素驱动（比如流动性宽松同时推高两只股票），也可能纯粹是巧合（样本够多总能找到高相关的两个序列）。相关系数是**描述**不是**解释**，用它来推断因果是量化新手的经典翻车点。

**警告二：相关性会在极端行情骤升。**

平时各自为政的资产，危机时刻会突然「一起跳崖」——流动性挤兑时所有人都卖一切能卖的东西，相关性瞬间冲向 +1。这意味着：**你按平时的相关矩阵算出来的分散效果，在最需要它的那天会失效**。做组合风控时，要用压力情景（比如危机时段的相关矩阵）单独再算一遍，别只信全样本均值。
