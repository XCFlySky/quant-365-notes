# ⑤ 避坑指南与实战规则

> 这些坑你不主动避，迟早会主动找上你。每个坑都附避坑动作，而不是只描述现象。

## 坑 1：把波动率当成「亏损概率」

- **现象**：「这股票年化波动 30%，是不是一年要亏 30%？」
- **真相**：这是最常见的误读。波动率只衡量散度（双向），不是衡量「会不会亏」。年化波动 30% 的股票同样可能涨 30%。
- **避坑动作**：想知道「最坏跌多少」，去看专门的指标——最大回撤、VaR、CVaR。波动率回答的是「平时多大震」，不是「会不会塌」。

## 坑 2：用月度数据估日度波动率，平方根位数错位

- **现象**：拿到月度波动率，直接除以 21（一个月约 21 个交易日）换算日波动率。
- **真相**：正确公式是 σ_daily ≈ σ_monthly / **√21**。错写成除以 21，日波动率会被算小 4–5 倍，后面所有 Sharpe 全错。
- **避坑动作**：能拿到日数据就用日数据，不要从粗粒度反推细粒度。换算永远带根号。

## 坑 3：看到历史波动率小就以为永远稳

- **现象**：「这资产过去几个月波动很小，很安全。」
- **真相**：波动率聚集意味着它会突然爆发。课程样本：2008 年 8 月之前几个月标普日波动只有 0.6–0.8%，9 月雷曼倒闭后立刻飙到 3–5%。按 8 月的历史波动率配仓位，9 月就会被扫地出门。
- **避坑动作**：实战看 **21 日滚动波动率 + VIX 当前水平**，不要只看长期均值——均值会掩盖聚集期。

## 坑 4：不年化就跨期比较

- **现象**：拿「日波动率 1%」跟「年化波动 20%」直接比，或者拿月波动率跟年波动率比。
- **真相**：风马牛不相及。不同时间尺度的波动率没有任何可比性。
- **避坑动作**：永远先把所有数字年化（× √N）再比较。这是最基本的金融素养——比较之前先统一单位，跟物理课一个道理。

## 坑 5：ddof 参数没显式写，小样本误差几个百分点

- **现象**：同一组数据，`np.std()` 和 `pd.Series.std()` 算出来不一样，不知道信哪个。
- **真相**：numpy 默认 `ddof=0`（除以 N），pandas 默认 `ddof=1`（除以 N−1）。30 天小样本里两者结果可以差 3–4%。
- **避坑动作**：研究代码里永远**显式写 ddof=1**（贝塞尔修正），或者全程统一用 pandas 的 `.std()`。出 paper、给客户看的数字，口径必须前后一致。

## 实战规则 SOP

| # | 规则 | 一句话理由 |
| --- | --- | --- |
| 01 | 看波动率前先确认是日 / 周 / 月 / 年化哪一种，不年化不比较 | 先统一单位再比较 |
| 02 | 永远用对数收益（Day 005 讲过）算波动率，简单收益只用于面向人的展示 | 对数收益可加，数学上才成立 |
| 03 | 样本方差除以 N−1（贝塞尔修正），numpy 写 ddof=1，pandas 默认就是 1 | 除 N 会系统性低估方差 |
| 04 | 年化：σ_ann = σ_daily × √252（美股），A 股可用 √244，加密可用 √365 | 方差线性可加，标准差按根号合并 |
| 05 | 实战看 21 日滚动波动率 + 长期均值，不要只看长期均值掩盖聚集期 | 大震后有余震，平静期可持续 |
| 06 | 高波动资产小仓位 + 短持有期，低波动资产可重仓 + 长持有 | 先有游戏难度，再谈下注大小 |
| 07 | 买入前心里默念：「年化波动 X%，意味着今年最坏可能跌 0.7–1×X%，我能不能扛？」 | 波动率是心理预演工具 |
