# ② Bootstrap 与金融数据的独立性陷阱

> 上一页说 CLT 需要「独立同分布」——可金融数据偏偏最爱违反这一条。这一页讲怎么补救。

## Bootstrap：把一份样本「复制」出一万份

1979 年，斯坦福统计学家 Bradley Efron 发明了 **Bootstrap（自助法）**。核心思想特别朴素：

> 你只有一组样本？那就把它当成「总体」，**有放回地**反复抽，抽出很多个「伪样本」，用这些伪样本的统计量分布来估计真实的不确定性。

为什么有效？因为每次重采样都在模拟「从真实总体再抽一次样」的过程。重采样 10000 次后，统计量（比如均值）的分布会逼近真实分布——背后撑腰的还是 CLT。

```text
95% 置信区间 = [ Bootstrap 均值的 2.5% 分位数, 97.5% 分位数 ]
```

### 金融里的典型用法

你的策略回测得到年化 15%，这一个数字靠谱吗？用 Bootstrap 重采样 10000 次，算出 95% 置信区间是 8% 到 22%：

- 区间窄 → 点估计可信；
- 区间太宽或包含负数 → 策略不够稳健，别急着上实盘。

> [!NOTE]
> **Bootstrap 是小样本的救命稻草。** 比如你只有 100 个交易日的收益，传统方法算不了置信区间，Bootstrap 把这 100 个数重采样 10000 次就能给你一个稳健的区间。

## 陷阱：金融数据不独立

CLT 和 Bootstrap 都暗中假设样本独立，但金融数据有三种「不老实」：

| 现象 | 白话解释 | 后果 |
| --- | --- | --- |
| 自相关 | 今天收盘和明天高度相关 | 有效样本量比看上去少 |
| 波动率聚集 | 大涨大跌爱扎堆出现 | 危机期数据「全军异常」，扭曲推断 |
| 系统性传染 | 极端事件一起爆 | 尾部比正态预测厚得多 |

课件给了一个直观数字（课程样本）：A 股**日收益**自相关系数约 0.05（看似很小），但**分钟数据**自相关会飙到 0.3 以上——频率越高，独立性越差，CLT 越不可靠。

> [!WARNING]
> 把 1000 个日收益当独立样本算统计量，如果其中混着一段金融危机，整段数据都是异常的，你的推断会被严重扭曲。

## 补救办法：让数据「更独立」

这不代表 CLT 在金融里完全失效，关键是主动改善数据质量：

1. **用低频数据**：周线/月线代替 tick 或分钟线，相关性天然更低；
2. **去趋势**：用差分或对数收益代替价格本身；
3. **分组聚合**：把不同股票的同期收益混在一起平均（下面细讲）。

## 分组聚合：CLT 的实战魔法

当单个资产不满足正态假设时，把多个资产的同期收益求平均。根据 CLT，只要这些资产**不完全同步波动**，聚合后的分布就会更接近正态。

具体做法（课程示例）：

- 你有 50 只股票的日收益，每只都不太正态；
- 计算「每日这 50 只股票的平均收益」，得到一个新序列；
- 这个新序列往往比任何单只股票更正态——极端个股收益被「平均掉」了。

课件给的数字：单股票日收益的偏度可能是 **1.0**（明显右偏），但 50 只等权组合的偏度能降到 **0.3**。

> [!IMPORTANT]
> **回测分析组合收益，而不是个股收益。** 这样你的 t 检验、Sharpe ratio 才站得住脚。一句话：多样性 → 正态性。

## 一页对照表

| 场景 | 能用 CLT/Bootstrap 吗 | 原因 |
| --- | --- | --- |
| 月线收益，n=60 | ✅ 基本可用 | 频率低，自相关弱 |
| 日收益，n=1000，含危机期 | ⚠️ 谨慎 | 危机段不独立，建议剔除或分段 |
| 分钟级高频数据 | ❌ 不行 | 自相关 0.3+，CLT 收敛太慢 |
| 50 只等权组合日收益 | ✅ 比个股好 | 聚合改善正态性 |

> [!TIP]
> Bootstrap 不能凭空创造信息：如果原始样本有偏差（比如只包含牛市），Bootstrap 只会重复这个偏差。**垃圾进，垃圾出。**
